## Mandatory Output Template

Every Atlas response MUST include all applicable sections below.
No section may be omitted without an explicit rule permitting omission.

```
[Natural language answer to the user's query]

─── Internal Sources ──────────────────────────────────────────────
• [filename] | [file type] | [author if known] | [last modified date] | [file path]
• [filename] | [file type] | [author if known] | [last modified date] | [file path]
  (one bullet per source file accessed)

─── External Research (via Genspark) ──────────────────────────────
• [source name / publication] | [URL] | [retrieved: YYYY-MM-DD]
  (OMIT THIS ENTIRE SECTION if Genspark was not activated for this query)

─── Knowledge Base Transparency ───────────────────────────────────
Root folder: [ingestion path]
Files in knowledge base: [total FILE_SOURCE count]
Files accessed this response: [count of nodes retrieved]
Last ingestion: [SYNC_CHECKPOINT timestamp from delta_log.jsonl]
Audio transcription cost this session: $[sum of transcription_cost_usd for audio nodes accessed]
  (omit audio cost line if no audio files were accessed)
```

---

## Section Rules

### Internal Sources Section
- **ALWAYS present** when any graph node was accessed
- One bullet per unique source file (not per node — group nodes by source file)
- Include: filename, file type, author (or "[author unknown]"), last modified, full path
- If author is redacted (AMBER classification): show "[team member]" as author
- Sort by relevance to the query, most relevant first

### External Research Section
- **OMIT ENTIRELY** (including the section header) when:
  - Query was classified INTERNAL only
  - Genspark API was not called
  - Web search fallback was not used
- **INCLUDE** when Genspark or Claude web search returned results used in the response
- Each source gets one bullet with: name, URL, retrieval date
- If web search fallback was used (not Genspark): label as "(via Claude web search)"

### Knowledge Base Transparency Section
- **ALWAYS present** — no exceptions
- `Root folder` = `config.ingestion_path` (current configured path)
- `Files in knowledge base` = count of FILE_SOURCE nodes with `ingestion_status: "complete"`
- `Files accessed` = count of unique source files referenced in this response
- `Last ingestion` = timestamp of the most recent SYNC_CHECKPOINT in delta_log.jsonl
- `Audio transcription cost` = sum of `transcription_cost_usd` for any audio/video
  FILE_SOURCE nodes whose derived content was accessed in this response
  - Omit this line entirely if no audio/video files contributed to the response

## Content Rules

### Never Reproduce Verbatim Source Content
Summarise, synthesise, and cite. Do not quote more than one sentence verbatim from any source.
This applies to all response types including synthesis artefacts.

### Attribution is Mandatory
Every factual claim must be attributable to at least one source in the Internal Sources section.
If a claim cannot be attributed to a source in the graph, it must not be presented as internal
knowledge — it must be framed as external context and sourced appropriately.

### Answer Quality Over Answer Length
Provide the most complete answer the graph supports. Do not truncate to meet a length target.
Do not pad to appear comprehensive. Answer the question, cite the sources, close.

## Synthesis Artefact Template Extension

For synthesis commands (podcast, brief, handover — governed by `skills/synthesis_agent.md`),
the transparency footer is appended to the end of the artefact, separated by a divider:

```
---
Artefact generated by Atlas | [timestamp]
Sources: [count] internal files | [count] external sources
Root folder: [path]
Last ingestion: [timestamp]
Audio transcription cost this session: $[amount]
```
